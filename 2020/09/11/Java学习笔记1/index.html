Java学习笔记1

发表于 2020-09-11

1.概述

编译

javac -d destdir srcFile

-d destdir 指定编译生成的字节码存放位置

destdir是本地磁盘的一个有效路径(可以一”.”表示当前路径)

srcFile是Java源文件所在位置(绝对/相对路径)

如javac -d . HelloWorld.java

运行

java Java类名

大部分时候Java源文件的主文件名(不包括扩展名)与类名相同,故Java类名形如文件名

如java HelloWorld

Java源文件命名

扩展名为.java

若源代码定义了一个public类(亦最多仅能有一个),则源文件主文件名必须与该类相同

无public类则文件名可以随意

Java严格区分大小写

2.数据类型和运算符

注释

  1. 单行注释 // 代码

  2. 多行注释 /* 代码 */

  3. 文档注释

    写好文档注释,利用javadoc工具生成api文档

    javadoc 选项 Java源文件|包

    常用选项

    -d<directory> 指定api文档的生成路径

    -windowtitle<text> 指定api文档浏览器窗口标题

    doctitle<html-code> 指定概述页面的标题(对多个包生成api文档才有概述页面)

    header<html-code> 指定一个html格式的文本,包含每个页面的页眉

    其他选项可通过javadoc -help 查看

    示例:javadoc -d apidoc -windowtitle 测试 -doctitle 学习 javadoc工具的测试API文档 -header 我的类 *Test.java

    *是通配符表示任何字符串; ?仅代表单个字符串，但此单字必须存在;

    上述示例即如JavadocTest.java和Test等文件;

    *.java即当前路径下所有java源文件

    javadoc标记

    • 出现在类或者接口文档注释中的

      @see:”参见”,用于指定交叉参考的内容

      @deprecated:不推荐使用的方法

      @author作者 @version源文件版本

    • 出现在方法或构造器文档注释中的

      @see @deprecated

      @param:方法的参数说明信息

      @return:方法的返回值说明信息

      @throws和@exception:抛出的异常

    • 出现在成员变量的文档注释中的

      @see @deprecated

    示例javadoc -d apidoc_directory -windowtitle 测试 -doctitle JAVADOC生成测试文档API -header 小爷的所有类 -author -version *.java

    javadoc默认不会提取@author和@version,需要在使用javadoc工具时指定-author和-version两个选项

    包描述文件

    有关包注释不在java源文件中,而是存放在一个标准的html5文件(一般命名为package.html并放在相应包文件夹)中,javadoc工具会提取该包描述文件中的<body/> 元素里的内容作为该包的描述信息

    示例:javadoc -d apidoc -windowtitle 测试 -doctitle 学习 javadoc工具的测试API文档 -header 我的类 -version -author lee yeeku (lee

    yeeku 是两个包名)

标识符与关键字

标识符由字母,数字,下划线(_)和美元符($)组成,数字不能打头,不能含空格,不能是关键字或三个特殊的(literal)直接量(true,false,null)

关键字都是小写的

null值的类型是空类型(null type),这种类型没有名称,故不能声明或转换成该类型,空引用(null)是null类型变量唯一的值,可转换成任何引用类型

  • 基本类型
    • boolean类型
    • 数值类型
      • 整数类型(包括char)
      • 浮点类型
  • 引用类型
    • 类
    • 接口
    • 数组
    • null类型

int是最常用的整数类型，因此在通常情况下，直接给出一个整数值默认就是int类型。除此之外，有如下两种情形必须指出。
如果直接将一个较小的整数值（在byte或short类型的表数范围内）赋给一个byte或short变量，系统会自动把这个整数值当成byte或者short类型来处理。
如果使用一个巨大的整数值（超出了int类型的表数范围）时，Java不会自动把这个整数值当成long类型来处理。如果希望系统把一个整数值当成long类型来处理，应在这个整数值后增加l或者L作为后缀。通常推荐使用L，因为英文字母l很容易跟数字1搞混。

如long a = 999999999999999;会报错,提示:” 错误: 过大的整数:”加上L则不会,当然,如果右边的数值较小(在int 的范围内,则不会报错,但Java还是把这个数值当作int类型处理,再自动转换为long类型)(也可以(long)进行强制类型转换)

整数值的四种表示方式:十进制,二进制(以0B或0b开头),八进制(以0开头),十六进制(以0x或0X开头,a~f不区分大小写)

int val = 0b二进制数 该二进制数为32位时最高位是符号位,若数后有l后缀,则第64位为符号位

byte binVal = (byte)0b11101001; 右边数值默认占32位,强制转换为byte类型时产生溢出,导致binval变成了-23(符号位变成了1)

long binVal = 0b1...29个0...11L; 右边本是占32位的负数,加入L后缀后变成64位,符号位为0,即变为正数了

字符型值的表示方式:

  • 单个字符,如’A’
  • 转义字符,如’\n’
  • Unicode值,形如’\uXXXX’,XXXX是十六进制正数

浮点数的表示方式:

  • 十进制数形式(必须包含小数点,否则处理为int类型)
  • 科学计数法形式,如5.12e2(即5.12*10^2^),5.12E2,512E2

浮点类型默认为double(64位),处理为float(32位)类型要在数值后紧跟f或F

三个特殊的浮点数值:正无穷大,负无穷大,非数.用于表示溢出和出错.

  • 使用一个正数除以0将得到正无穷大
  • 使用一个负数除以0将得到负无穷大
  • 只有浮点数除以0才可以得到正无穷大或负无穷大，因为Java语言会自动把和浮点数运算的0（整数）当成0.0（浮点数）处理。
  • 浮点数对0或0.0求余得到非数:NaN
  • 如果一个整数值除以0，则会抛出一个异常ArithmeticException：/byzero（除以0异常）
  • 0.0除以0.0或对一个负数开方将得到一个非数。
  • 0或0.0对零以外的任何数求余都将得到0或0.0
  • 正无穷大通过Double或Float类的POSITIVEINFINITY表示
  • 负无穷大通过Double或Float类的NEGATIVEINFINITY表示
  • 非数通过Double或Float类的NaN表示。
  • 所有的正无穷大数值都是相等的，所有的负无穷大数值都是相等的；而NaN不与任何数值相等，甚至和NaN都不相等。

Java 7 后可通过在数值中使用下划线直观分辨数值的位数

如 0B1000_0000_0000 3.14_15_92

类型转换

  • 表数范围小的可向表数范围大的进行自动类型转换

任何基本类型的值与字符串值进行连接运算时将自动转换为字符串类型,因此可与空字符串””连接使得基本类型转换为字符串类型

3+4+”hello”实际是7+”hello”==>”7hello”

“hello”+3+4实际是”hello34”

  • 表数范围大向表数范围小的转换必须进行==强制类型转换==,运算符圆括号()

强制类型转换可能产生溢出(如8位的byte类型向32位的int类型转换时)

如float a = 5.6报错,因为5.6是double类型,改成float a = (float)5.6则正确

  • String类型转换为基本类型使用基本类型的包装类的parseXxx(String str)静态方法

  • 一个算术表达式含多个基本类型的值时会自动提升表达式类型(按以下原则)

    1. byte,short,char===>int
    2. 整体提升到最高等级操作数的类型

    'a'+7+"helllo"中’a’自动提升到int型,结果为97+7+”hello”==>”104hello”

    字符串直接量

    程序第一次使用某个字符串直接量时，Java会使用常量池（constant pool)来缓存该字符串直接量，如果程序后面的部分需要用到该字符串直接量时，Java会直接使用常量池中的字符串直接量.

    常量池（constant pool）指的是在编译期被确定，并被保存在已编译的.class文件中的一些数据。它包括关于类、方法、接口中的常量，也包括字符串直接量

自增自减

++放在左边，则先把操作数加1，然后才把操作数放入表达式中运算；如果把++放在右边，则先把操作数放入表达式中运算,再加一

赋值运算符

支持连续赋值

位运算符

  • 按位与&
  • 按位或|
  • 按位非~
  • 按位异或^
  • 左移运算符<<
  • 右移运算符>>
  • 无符号右移运算符>>>

一般只操作整数类型

低于int类型总是先自动转换为int类型再移位

对于int类型a>>b,当b>32时,系统用b对32求余,所得结果为移位位数,

a>>33等价于a>>1(long则是64)

左移n位相当于乘2的n次方,右移n位相当于除以2的n次方

逻辑运算符

  • 短路与&&
  • 不短路与&
  • 短路或 ||
  • 不短路或 |
  • 非 !
  • 异或 ^(两个操作数不同时返回true,相同返回false)

短路特性:&总会计算前后两个操作数,短路&&先计算左边,若为false,直接返回false,不会计算右边(或运算类似)

3. 流程控制与数组

switch语句

switch语句由一个控制表达式和多个case标签组成，和if语句不同的是，switch语句后面的控制表达式的数据类型只能是byte、short、char、int四种整数类型，枚举类型和java.lang.==String类型（从Java7才允许）==，不能是boolean类型。

每个case后代码块需加上break;语句否则后面的语句都会执行

for循环

对于for循环而言，循环条件总比循环体要多执行一次，因为最后一次执行循环条件返回false，将不再执行循环体。值得指出的是，for循环的循环迭代语句并没有与循环体放在一起，因此即使在执行循环体时遇到continue语句结束本次循环，循环迭代语句也一样会得到执行。

标签

后面带有冒号的标识符,如:

outer:

标签只有放在循环语句前才有作用

可用于break或continue

数组

数组也是一种数据类型(引用类型,如int[])

定义

定义数组仅仅定义了一个引用变量,还不能指定数组长度.

初始化

即为数组的数组元素分配内存空间并赋初值

数组的初始化有如下两种方式(不要同时使用)

  • 静态初始化：初始化时由程序员显式指定每个数组元素的初始值，由系统决定数组长度。

    arrayName = new type[]{element1,element2,element3,...};

    数组元素值必须是type或其子类的实例

    简化写法(定义同时初始化才可使用)

    type[] arrayName = {element1,element2,element3,...};

    如int[] a = {5,6,7,9};

  • 动态初始化：初始化时程序员只指定数组长度，由系统为数组元素分配初始值。

    arrayName = new type[ length ];

访问数组元素

索引值小于0或大于等于数组长度时编译不会报错,运行时会出现数组索引越界异常

foreach循环

foreach的循环变量相当于临时变量,不是数组元素,只是保存了数组元素的值,故不能在foreach循环中通过赋值改变数组元素的值.

数组在内存中

数组引用变量保存在栈(stack)内存中,实际的数组对象(数组元素)保存在堆(heap)内存中,堆内存中数组没有引用变量指向时会被回收.

对于引用类型数组,假如定义了一个对象数组,初始化后,数组的引用变量和对象的引用变量保存在栈内存中, 该数组的数组元素也是引用,保存在堆内存中,并指向堆内存中保存的对象

多维数组

多维数组实际是数组元素为引用,不是真正的多维数组.

定义

type[][] arrName //实质还是一维数组,数组元素是指向一维数组的引用

初始化

arrName = new type[ length ][ ] //当成元素类型为type[]的一维数组的初始化,之后再对每个数组元素进行数组初始化

要把一维数组扩展成二维,三维…,可定义一个Object[]类型数组,使元素指向另一个Object[]类型数组,则可无限扩展下去

亦可以同时初始化多个维数

int[][] b = new int[3][4];
